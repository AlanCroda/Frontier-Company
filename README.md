# Frontier Company

Frontier Company is a compact, pausable, systems-driven strategy game being developed in Unity by a solo developer. The player builds and controls a small organization operating inside a changing regional ecosystem.

This repository is in its initial production-foundation stage. The current priority is establishing lightweight project context for design work and future Codex implementation work without committing to detailed gameplay systems too early.

## Project posture

Frontier Company prioritizes:

- ownership and company identity;
- meaningful risk;
- systemic interaction;
- contextual strategy;
- emergent consequences;
- bounded campaigns;
- finishable scope;
- early testability.

## Core documentation

- `AGENTS.md` — working instructions for Codex and future agents.
- `docs/vision.md` — stable project identity, design pillars, scope principles, and non-goals.
- `docs/current-state.md` — factual snapshot of what currently exists.
- `docs/decisions.md` — accepted, open, and rejected project decisions.
- `docs/milestones.md` — lightweight playable-chunk milestone direction.

## Development status

The repository contains an early Unity project scaffold. Detailed Unity setup guidance should wait until the project has been imported through Unity Hub and Unity has completed its required project actions.

## How to start

1. Read `docs/vision.md` for the project constraints.
2. Read `docs/current-state.md` for the current factual state.
3. Check `docs/decisions.md` before making design or architecture assumptions.
4. Check `docs/milestones.md` for the current playable-chunk direction.
5. Follow `AGENTS.md` when using Codex or another agent in this repository.
